`def decay_confidence(current: float, days_since_update: float, is_human_confirmed: bool, decay_weekly: float = 0.01, decay_confirmed_monthly: float = 0.005) -> float`:
- If is_human_confirmed: rate_per_day = decay_confirmed_monthly / 30.0
- Else: rate_per_day = decay_weekly / 7.0
- decayed = current * ((1.0 - rate_per_day) ** days_since_update)
- Return max(0.0, decayed)

`def reinforce_confidence(current: float, evidence_weight: float = 0.1) -> float`:
- Return min(1.0, current + evidence_weight)

`def resolve_conflict(fact_a_conf: float, fact_a_sources: int, fact_a_ts: int, fact_b_conf: float, fact_b_sources: int, fact_b_ts: int, escalate_threshold: float = 0.1, fact_a_context: str | None = None, fact_b_context: str | None = None) -> tuple[str, str]`:
- score_a = fact_a_conf * 0.6 + min(1.0, fact_a_sources / 5.0) * 0.3 + (1.0 if fact_a_ts > fact_b_ts else 0.0) * 0.1
- score_b = fact_b_conf * 0.6 + min(1.0, fact_b_sources / 5.0) * 0.3 + (1.0 if fact_b_ts > fact_a_ts else 0.0) * 0.1
- diff = abs(score_a - score_b)
- **KEEP branch:** If fact_a_context and fact_b_context differ and both have confidence >= 0.3: return ("KEEP", f"Facts coexist under different conditions: A={fact_a_context}, B={fact_b_context}"). This handles cases where both facts are valid but apply to different conditions (e.g., enum value means different things in different contexts).
- **SPLIT branch:** If fact_a_context and fact_b_context are both present and non-empty but partially overlap: return ("SPLIT", f"Facts split into sub-contexts: A={fact_a_context}, B={fact_b_context}"). This handles cases where a fact should be subdivided by context (e.g., column meaning varies by business unit).
- If diff < escalate_threshold: return ("ESCALATE", f"Score difference {diff:.3f} < threshold {escalate_threshold}")
- If score_a > score_b: return ("SUPERSEDE_B", f"Fact A score {score_a:.3f} > Fact B {score_b:.3f}")
- Else: return ("SUPERSEDE_A", f"Fact B score {score_b:.3f} > Fact A {score_a:.3f}")

`def compute_sp_reliability(conn: sqlite3.Connection, proc_id: int, now_ts: int) -> float`:
- score = 0.5 (baseline)
- Query current_sp_reliability WHERE procedure_id = proc_id for has_dynamic_sql, partial_ast, parse_quality
- If no row found, return 0.5
- If has_dynamic_sql: score -= 0.2
- If partial_ast: score -= 0.05
- Query current_db_procedures WHERE id = proc_id for valid_from_ts; if (now_ts - valid_from_ts) < 30 * 86400: score += 0.1
- Query COUNT(DISTINCT caller_id) FROM current_sp_call_chains WHERE callee_id = proc_id; score += min(0.25, caller_count * 0.05)
- Query COUNT(*) FROM current_agent_results ar JOIN current_agent_tasks at ON ar.task_id = at.id WHERE at.gap_id IN (SELECT id FROM current_knowledge_gaps WHERE entity_id = proc_id) AND ar.approved = 0; score -= min(0.5, contradictions * 0.1)
- Return max(0.0, min(1.0, score))

`def count_independent_sources(conn: sqlite3.Connection, entity_type: str, entity_name: str, attribute: str) -> int`:
- Per D-16: simple source counting. Each SP is one source.
- For enum_values: COUNT(DISTINCT source_procedure) FROM current_enum_values WHERE table_name || '.' || column_name = entity_name
- For other types: return 1 as default

`def find_existing_fact(conn: sqlite3.Connection, entity_type: str, entity_name: str, attribute: str) -> dict | None`:
- Parse entity_name to extract table_name and column_name (split on '.')
- Based on attribute type, query the appropriate Phase 2 table:
  - attribute="enum_label": query current_enum_values WHERE table_name=? AND column_name=? -- return dict with id, value (=enum_label), confidence, source_procedure, human_confirmed, recorded_at_ts
  - attribute="description": query current_db_tables or current_db_procedures WHERE name=? -- return dict with id, value (=description), confidence (default 0.5)
  - attribute="bitmask_label": query current_bitmask_definitions
  - attribute="alias": query current_column_aliases
  - Default: return None (unknown attribute type)

`def classify_update(conn: sqlite3.Connection, entity_type: str, entity_name: str, attribute: str, new_value: str, new_confidence: float) -> tuple[UpdateOp, dict | None]`:
- Call find_existing_fact(conn, entity_type, entity_name, attribute)
- If no existing: return (UpdateOp.ADD, None)
- If existing["value"] == new_value:
  - If abs(existing["confidence"] - new_confidence) < 0.05: return (UpdateOp.NOOP, existing)
  - Else: return (UpdateOp.REINFORCE, existing)
- Else: return (UpdateOp.CONFLICT, existing)

`def apply_findings(conn: sqlite3.Connection, gap: GapRecord, findings: AgentFindings, now_ts: int, now_iso: str) -> dict`:
- results = {"add": 0, "reinforce": 0, "conflict": 0, "noop": 0, "escalated": 0}
- For each item in findings.items:
  - op, existing = classify_update(conn, item.entity_type, item.entity_name, item.attribute, item.value, item.confidence)
  - If op == ADD: insert new bi-temporal row using `_write_to_target_table()` helper. Target table mapping by FindingItem.attribute: attribute="enum_label" -> INSERT INTO enum_values (table_name, column_name, enum_value, enum_label, confidence, source_procedure, human_confirmed, + 8 bi-temporal cols); attribute="description" -> UPDATE via invalidate+insert on db_tables or db_procedures; attribute="bitmask_label" -> INSERT INTO bitmask_definitions; attribute="alias" -> INSERT INTO column_aliases. This is the LEARN-05 enum discovery write path: when Research Agent produces FindingItem(attribute="enum_label", value="Active") for an unlabeled_enum gap, apply_findings ADDs it to enum_values. results["add"] += 1
  - If op == REINFORCE: compute new_conf = reinforce_confidence(existing["confidence"]). Invalidate old row (set invalidated_at), insert new row with updated confidence. results["reinforce"] += 1
  - If op == CONFLICT: call resolve_conflict(). If SUPERSEDE: invalidate old, insert new in target table. If KEEP: insert both as separate rows (different conditions). If SPLIT: create sub-context rows. If ESCALATE: create new gap of type "escalated_conflict". Log rationale. results["conflict"] += 1
  - If op == NOOP: results["noop"] += 1
- conn.commit()
- Return results

`def mark_gap_resolved(conn: sqlite3.Connection, gap_id: int, now_ts: int, now_iso: str) -> None`:
- UPDATE current_knowledge_gaps row: set status='resolved', invalidated_at=now_iso, invalidated_at_ts=now_ts for the existing row
- Insert new bi-temporal row with status='resolved', resolution_notes filled

`def bump_attempt_count(conn: sqlite3.Connection, gap_id: int, learning_config: LearningConfig, now_ts: int, now_iso: str) -> None`:
- Read current attempt_count from gap
- new_count = attempt_count + 1
- If new_count >= learning_config.max_attempts_before_permanent: set status='permanent'
- Else: compute cooldown = learning_config.cooldown_hours[min(new_count-1, len-1)] * 3600; set cooldown_until_ts = now_ts + cooldown
- Invalidate old row, insert updated bi-temporal row
