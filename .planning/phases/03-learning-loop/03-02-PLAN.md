Create `db_wiki/learning/gap_detector.py` with:

Import `sqlite3`, `time`, `logging`, `datetime`, `collections.defaultdict` and from `db_wiki.learning.models` import `GapInfo`.

Implement these 12 detection functions, each taking `conn: sqlite3.Connection` and returning `list[GapInfo]`:

1. `detect_unlabeled_enums(conn)` -- Query current_enum_values WHERE enum_label IS NULL OR enum_label = '', GROUP BY table_name, column_name. Severity 0.7.
2. `detect_orphan_tables(conn)` -- Query current_db_tables WHERE NOT EXISTS in current_db_relationships (source_id or target_id). Severity 0.5.
3. `detect_missing_joins(conn)` -- Query current_db_relationships WHERE relationship_type='joins_with' AND NOT EXISTS (fk_declared or fk_inferred for same source_id/target_id pair). Severity 0.6.
4. `detect_stale_facts(conn, staleness_days=90)` -- Query current_enum_values WHERE confidence < 0.7 AND recorded_at_ts < threshold. Severity 0.4.
5. `detect_alias_clusters(conn)` -- Query current_column_aliases GROUP BY table_name, column_name HAVING COUNT(*) > 1. Severity 0.3.
6. `detect_incomplete_state_machines(conn)` -- Query current_state_transitions, group by (table_name, column_name), flag groups with < 2 distinct state values. Severity 0.6.
7. `detect_unresolved_calls(conn)` -- Query current_sp_call_chains WHERE is_resolved = 0, JOIN current_db_procedures. Severity 0.5.
8. `detect_low_confidence_facts(conn, threshold=0.4)` -- UNION query across current_enum_values, current_bitmask_definitions, current_column_aliases WHERE confidence < threshold. Severity 0.4.
9. `detect_cross_sp_contradictions(conn)` -- Self-join current_enum_values where same (table_name, column_name, enum_value) has different enum_labels. Severity 0.8.
10. `detect_missing_fks(conn)` -- Query current_db_columns WHERE column_name LIKE '%_id' OR '%_code', is_primary_key=0, AND NOT EXISTS FK relationship. Severity 0.6.
11. `detect_coverage_gaps(conn)` -- UNION query: current_db_tables WHERE description IS NULL, current_db_procedures WHERE description IS NULL. Severity 0.3.
12. `detect_pattern_anomalies(conn)` -- Query current_sp_reliability WHERE parse_quality < 0.8 OR is_degraded = 1, JOIN current_db_procedures. Severity 0.4.

All SQL uses `?` parameterized queries where parameters exist. Entity names use stable format: `f"{table_name}.{column_name}"` for column gaps, `f"{source_table}_to_{target_table}"` for relationship gaps (join to get table names, not IDs).

Implement `detect_all_gaps(conn: sqlite3.Connection, now_ts: int, now_iso: str) -> list[GapInfo]` that calls all 12 rules and concatenates results. Log count per rule at DEBUG level.

Implement `upsert_gaps(conn: sqlite3.Connection, gaps: list[GapInfo], now_ts: int, now_iso: str) -> list[int]`:
- For each gap, check current_knowledge_gaps for existing row matching (gap_type, entity_name)
- If status='open' or 'investigating': skip (already tracked), return existing id
- If status='permanent': skip entirely, return None
- If status='resolved' and cooldown_until_ts > now_ts: skip (still cooling down)
- If status='resolved' and cooldown_until_ts <= now_ts or NULL: insert new bi-temporal row (re-open)
- If no existing: insert new row with all bi-temporal columns set (valid_from=now_iso, valid_from_ts=now_ts, recorded_at=now_iso, recorded_at_ts=now_ts, valid_until=NULL, invalidated_at=NULL)
- Return list of inserted/existing gap IDs

Create `db_wiki/learning/gap_scorer.py` with:

Import from `db_wiki.learning.models` GapInfo, GapRecord. Import from `db_wiki.core.config` LearningGapWeightsConfig, LearningConfig.

`SOLVABLE_WITH_SAMPLING` = set of gap_types that benefit from live DB: {"unlabeled_enum", "missing_fk", "stale_fact", "low_confidence_fact", "alias_cluster"}

`def score_gap(conn: sqlite3.Connection, gap: GapInfo, weights: LearningGapWeightsConfig, gap_recorded_at_ts: int = 0) -> float`:
- severity = gap.severity (already 0.0-1.0)
- connectivity: if gap.entity_id, call `from db_wiki.graph.bfs import bfs_graph; hops = bfs_graph(conn, gap.entity_id, max_depth=2); connectivity = min(1.0, len(hops) / 20.0)` else 0.0. Wrap in try/except returning 0.0 on failure.
- query_frequency = 0.5 (neutral default -- Phase 4 adds query logs)
- staleness: `age_days = (int(time.time()) - gap_recorded_at_ts) / 86400; staleness = min(1.0, age_days / 30.0)` -- for new gaps age is 0 so staleness is 0
- solvability: 0.7 if gap.gap_type in SOLVABLE_WITH_SAMPLING else 0.3
- Return `weights.severity * severity + weights.connectivity * connectivity + weights.query_frequency * query_frequency + weights.staleness * staleness + weights.solvability * solvability`

`def score_and_prioritize(conn: sqlite3.Connection, config: LearningConfig) -> list[GapRecord]`:
- Query current_knowledge_gaps WHERE status='open' to get all open gaps
- For each, compute score_gap() and UPDATE priority_score in the table
- Sort by priority_score DESC, return top config.max_gaps_per_run as GapRecord list

`def get_eligible_gaps(conn: sqlite3.Connection, max_gaps: int, now_ts: int) -> list[GapRecord]`:
- Query current_knowledge_gaps WHERE status = 'open' AND (cooldown_until_ts IS NULL OR cooldown_until_ts <= now_ts) ORDER BY priority_score DESC LIMIT max_gaps
- Return as list[GapRecord]
