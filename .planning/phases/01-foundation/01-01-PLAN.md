1. Create `db_wiki/core/config.py` with:

   ```python
   from pathlib import Path
   from pydantic import BaseModel

   class StorageConfig(BaseModel):
       path: str = ".db-wiki"

   class DatabaseConfig(BaseModel):
       connection_string: str | None = None
       timeout_seconds: int = 30

   class IngestConfig(BaseModel):
       max_file_size_mb: int = 50

   class DBWikiConfig(BaseModel):
       storage: StorageConfig = StorageConfig()
       database: DatabaseConfig = DatabaseConfig()
       ingest: IngestConfig = IngestConfig()
   ```

   Add `load_config(store_path: Path) -> DBWikiConfig`:
   - Resolves `store_path / "config.yaml"` to absolute path
   - If file exists, reads it with `yaml.safe_load()`, passes dict to `DBWikiConfig(**data)` (or `DBWikiConfig.model_validate(data)`)
   - If file doesn't exist or is empty, returns `DBWikiConfig()` with defaults
   - Per D-07: store_path is configurable, defaults to `.db-wiki`

   Add `write_default_config(store_path: Path) -> None`:
   - Creates `store_path / "config.yaml"` with the default config values
   - Uses `yaml.dump(config.model_dump())` for clean YAML output
   - Creates parent directories if needed

   NOTE: Use plain pydantic BaseModel + yaml.safe_load instead of pydantic-settings YamlConfigSettingsSource — simpler, avoids the path resolution pitfall (Pitfall 3 from RESEARCH.md), and pydantic-settings YAML source has uncertain import paths (Open Question 3). The config file is small enough that manual YAML loading is cleaner.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system -> store.py | User-supplied --store-path crosses into SQLite file creation |
| YAML file -> config.py | User-editable config.yaml loaded and parsed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | store.py open_store | mitigate | Resolve db_path to absolute with Path.resolve() before sqlite3.connect(); prevent path traversal |
| T-01-02 | Tampering | config.py load_config | mitigate | Use yaml.safe_load() (never yaml.load()); pydantic validates all types after loading |
| T-01-03 | Denial of Service | config.py load_config | accept | Config files are small and local-only; no network exposure. User controls their own files. |
| T-01-04 | Information Disclosure | store.py knowledge.db | accept | Single-user local tool; DB file permissions managed by OS. No network-accessible path. |
</threat_model>
