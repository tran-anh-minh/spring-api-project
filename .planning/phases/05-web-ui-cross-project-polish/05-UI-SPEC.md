# Phase 5 — UI Design Contract

> Visual and interaction contract for the web UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — plain HTML served as starlette StaticFiles |
| Preset | not applicable |
| Component library | none — CDN libraries only (vis.js, Chart.js) |
| Icon library | none — Unicode/emoji glyphs for status indicators only |
| Font | system-ui, -apple-system, sans-serif (no web font load) |

**Rationale (source: RESEARCH.md Standard Stack):** This is a Python project. The web UI is a single static HTML page served from `db_wiki/web/static/`. React/shadcn is not applicable. All interactivity is delivered by vis-network@10.0.2 and chart.js@4.5.1 loaded from CDN. No build step.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Toolbar button padding, chip spacing |
| md | 16px | Panel content padding, card inner padding |
| lg | 24px | Section divider gap, side panel header padding |
| xl | 32px | Side panel width offset when open |
| 2xl | 48px | Dashboard chart top margin |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Side panel width: 360px (fixed, not a spacing token — this is a layout dimension per D-02)
- Graph canvas: fills viewport minus panel width when panel is open
- Toolbar height: 48px (touch-friendly minimum for icon buttons)

## Typography

All type is set in the system font stack. No font loading.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.4 |
| Heading | 16px | 600 | 1.3 |
| Display | 20px | 600 | 1.2 |

Usage mapping:
- Body (14px/400): Side panel wiki content, dashboard description text
- Label (12px/600): Node type badges, edge type tags, legend items, stat values in dashboard
- Heading (16px/600): Side panel entity name, dashboard section titles
- Display (20px/600): Page title in toolbar ("DB Wiki"), dashboard coverage % metric

## Color

This is a dark-themed data tool — matches developer expectations for a graph visualization interface.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #1a1a2e | Main page background, graph canvas background |
| Secondary (30%) | #16213e | Side panel background, toolbar background, dashboard card backgrounds |
| Accent (10%) | #0f3460 | Active toolbar buttons, selected node ring, focus states, primary CTA buttons |
| Text primary | #e0e0e0 | All body text, headings |
| Text secondary | #9ca3af | Label text, placeholder text, disabled states |
| Destructive | #dc2626 | Destructive action confirmation only — no other use |

Accent reserved for:
1. Selected/active node outline in vis.js graph
2. Active toolbar button background
3. "Export All" primary action button
4. Focused form input border in search field
5. Dashboard metric sparkline color

vis.js node background colors by entity type — used in normal view mode:

| Entity Type | Color | Rationale |
|-------------|-------|-----------|
| table | #3b82f6 | Blue — data containers, primary entities |
| stored_procedure | #8b5cf6 | Purple — logic layer |
| column | #10b981 | Green — leaf nodes, attributes |
| index | #f59e0b | Amber — structural metadata |
| constraint | #6b7280 | Gray — secondary metadata |

### Confidence Heat-Map Mode (source: CONTEXT.md D-03)

Toggle: "Type View" | "Confidence View" (button in toolbar)

In Confidence View, node background color overrides type color:

| Confidence Range | Color | Opacity |
|-----------------|-------|---------|
| 0.8–1.0 | #22c55e | 1.0 |
| 0.6–0.79 | #84cc16 | 0.9 |
| 0.4–0.59 | #eab308 | 0.75 |
| 0.2–0.39 | #f97316 | 0.6 |
| 0.0–0.19 | #ef4444 | 0.45 |

Edge opacity scales linearly with confidence: `opacity = max(0.15, confidence_score)`

### Gap Highlighting (source: REQUIREMENTS.md UI-06, CONTEXT.md D-03)

Gaps are nodes present in `knowledge_gaps` table with no resolution:
- `borderDashes: [6, 3]` in vis.js node options (dashed border)
- CSS keyframe pulse animation on the node border: 0.8s ease-in-out infinite, border-color alternates between #ef4444 and transparent

## Layout Contract

### Graph Page (`/`)

```
┌────────────────────────────────────────────────────────────────────────┐
│ Toolbar (48px height, secondary bg)                                    │
│  [DB Wiki]  [Search input]  [Type View | Confidence View]  [Export All]│
└────────────────────────────────────────────────────────────────────────┘
┌──────────────────────────────────┬─────────────────────────────────────┐
│                                  │  Side Panel (360px, secondary bg)   │
│  Graph Canvas (vis.js Network)   │  [Entity Name heading]              │
│  Full viewport height minus      │  [Entity type label badge]          │
│  toolbar                         │  [L1 wiki content — body text]      │
│  Canvas shrinks when panel open  │  [L2 wiki detail — body text]       │
│                                  │  [Confidence score label]           │
│                                  │  [Related gaps — label badges]      │
│                                  │  [Close button — bottom right]      │
│                                  │                                     │
└──────────────────────────────────┴─────────────────────────────────────┘
```

Panel open behavior (source: CONTEXT.md D-02): CSS `transition: width 0.25s ease` on graph container. Panel slides in from right. Graph reflows to fill remaining width. No overlay — graph remains interactive while panel is open.

### Dashboard Page (`/dashboard`)

```
┌──────────────────────────────────────────────────────────────────────┐
│ Toolbar (same as graph page, 48px)                                   │
└──────────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────┐
│  Stat Row (3 cards, equal width, secondary bg, md padding)          │
│  [Schema Coverage %] | [Open Gaps N] | [Conflicts N]                │
└─────────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────┐
│  Charts Row (2 charts side-by-side, Chart.js)                       │
│  [Coverage Over Time — line chart] | [Gap Burndown — bar chart]     │
└─────────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────┐
│  Top Gaps Table (full width)                                        │
│  gap_type | description | severity | attempts | last_attempted      │
└─────────────────────────────────────────────────────────────────────┘
```

### vis.js Layout Algorithm (source: CONTEXT.md Claude's Discretion)

- Layout: `forceAtlas2Based` (hierarchical clustering, prevents overlapping, good for schema graphs)
- Physics: enabled during initial layout, disabled after stabilization (`stabilization.enabled: true`, `stabilization.iterations: 200`)
- Edge smooth: `{type: 'dynamic'}` for curved edges showing edge direction
- Node shape: `dot` for tables, `square` for SPs, `diamond` for columns, `triangle` for indexes
- Clustering: group nodes with >10 edges into cluster with label "N tables" to prevent overload (vis.js cluster API)

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Node click | single click on graph node | Fetch `/api/wiki?entity_id=X`, open side panel with L1/L2 content |
| Node expand | double-click on graph node | Fetch `/api/graph?node=X&depth=1`, add new neighbor nodes to canvas |
| Search | type in toolbar search input (debounced 300ms) | Fetch `/api/search?q=...`, highlight matching nodes, pan to first result |
| Filter clear | Escape key in search input | Clear highlight, return to full graph view |
| Panel close | click "Close" button or Escape key | Slide panel out, graph expands back to full width |
| View toggle | click "Confidence View" / "Type View" | Re-render all node colors using confidence scale or type color map |
| Zoom/pan | mouse wheel + drag | Native vis.js — no custom code needed |
| Export All | click "Export All" toolbar button | POST `/api/export`, show inline progress indicator, download link on complete |
| Dashboard nav | click "Dashboard" link (if present in toolbar) | Navigate to `/dashboard` page |

## API Endpoints (consumed by static HTML)

| Endpoint | Method | Response Shape | Page |
|----------|--------|---------------|------|
| `/api/graph` | GET | `{nodes: [{id, label, type, confidence}], edges: [{from, to, type, confidence}]}` | Graph |
| `/api/graph?node=X&depth=N` | GET | Same shape, subgraph centered on X | Graph |
| `/api/wiki?entity_id=X` | GET | `{name, type, l1_content, l2_content, confidence, gaps: []}` | Graph side panel |
| `/api/search?q=...` | GET | `{results: [{entity_id, name, type, score}]}` | Graph toolbar |
| `/api/dashboard` | GET | `{coverage_pct, gap_count, conflict_count, history: [], top_gaps: []}` | Dashboard |
| `/api/export` | POST | `{status, export_path}` | Graph toolbar |
