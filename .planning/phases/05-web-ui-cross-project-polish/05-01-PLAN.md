4. Create db_wiki/web/routes.py with 6 async route handlers. Each wraps blocking DB calls with anyio.to_thread.run_sync() (per RESEARCH anti-pattern: blocking event loop):

graph_api(request) -> JSONResponse:
  - If no "node" query param: initial load — query current_db_tables for all tables, current_db_relationships for FK edges
  - Nodes format: {"id": "t_{id}", "label": table_name, "title": description, "group": "table", "opacity": max(0.3, confidence or 1.0), "confidence": confidence or 1.0}
  - Check knowledge_gaps table to set "borderDashes": [6, 3] and "is_gap": true on gap nodes
  - Edges format: {"from": "t_{source_id}", "to": "t_{target_id}", "label": relationship_type, "confidence": confidence or 1.0}
  - If "node" param present: validate depth <= 5 (T-05-01 per VALIDATION.md), call bfs_graph(conn, int(node_id), max_depth=depth), return BFS results as nodes+edges
  - Return JSONResponse({"nodes": [...], "edges": [...]})

wiki_api(request) -> JSONResponse:
  - Requires entity_id query param (return 400 if missing)
  - Look up entity type from current_db_tables or current_db_procedures by id
  - Call get_wiki_page(conn, entity_type, entity_id, "L1") and get_wiki_page(conn, entity_type, entity_id, "L2")
  - Query knowledge_gaps for this entity
  - Query confidence from the entity's current_* view
  - Return JSONResponse({"name": ..., "type": ..., "l1_content": ..., "l2_content": ..., "confidence": ..., "gaps": [...]})

search_api(request) -> JSONResponse:
  - Requires "q" query param (return 400 if missing)
  - Call hybrid_search(conn, q, config.embedding, limit=20)
  - Return JSONResponse({"results": [{"entity_id": id, "name": name, "type": type, "score": score}]})
  - Look up entity names from current_db_tables/current_db_procedures after hybrid_search returns IDs

dashboard_api(request) -> JSONResponse:
  - Query current_db_tables COUNT for total tables
  - Query current_db_columns COUNT for total columns
  - Query knowledge_gaps WHERE resolution IS NULL for gap_count
  - Query knowledge_gaps WHERE resolution = 'conflict' or similar for conflict_count
  - Coverage = (tables with confidence >= 0.5) / total tables * 100
  - Query knowledge_gaps grouped by gap_type for top_gaps list (top 10 by severity)
  - Return JSONResponse({"coverage_pct": float, "gap_count": int, "conflict_count": int, "top_gaps": [...], "history": []})

export_api(request) -> JSONResponse:
  - POST handler. Deferred to Plan 04 for actual implementation.
  - For now: return JSONResponse({"status": "not_implemented", "export_path": ""}, status_code=501)

All route functions accept (conn, config) via closure from create_web_app. Use functools.partial or define them inside create_web_app.

| T-05-04 | Spoofing | /api/graph node param | mitigate | Parse node param as int; return 400 on non-integer to prevent SQL injection via node ID |
