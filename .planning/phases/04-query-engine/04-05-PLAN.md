| CLI user input -> query pipeline | User-typed questions and entity names enter the pipeline |
| CLI -> live DB (ask --execute) | Generated SQL crosses to production database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Injection | cli/app.py:ask | mitigate | Same mitigations as MCP ask tool — pipeline validates SQL via sqlglot qualify(). Executor rejects non-SELECT. --execute flag requires explicit user action. |
| T-04-15 | Injection | cli/app.py:define-metric | mitigate | SQL expression validated by resolver.define_metric() with sqlglot.parse_one() and keyword blacklist before storage. |
| T-04-16 | Information Disclosure | cli/app.py:coverage,data-quality | accept | Intentionally exposes knowledge store statistics to the local CLI user. No network exposure. No PII in schema metadata. |
</threat_model>
