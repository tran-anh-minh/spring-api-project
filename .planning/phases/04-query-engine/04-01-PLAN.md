       - `@dataclass ResolvedEntity`: entity_type (str), entity_id (int), name (str), score (float)
       - `@dataclass JoinStep`: from_table (str), to_table (str), join_type (str), edge_type (str), from_column (str|None), to_column (str|None)

Filter results for paths reaching end_table_id. For each edge in the path, look up source/target column names from `current_db_relationships` where available. Return list of JoinStep.
       - `def define_metric(conn, name: str, sql_fragment: str, source_tables: list[str], description: str | None = None) -> int`: Validate sql_fragment with `sqlglot.parse_one(sql_fragment, dialect="tsql")` — reject if ParseError. Check fragment does not contain `;`, `DROP`, `INSERT`, `DELETE`, `EXEC`, `UPDATE` keywords (case-insensitive). Insert into derived_metrics with bi-temporal timestamps (valid_from=now, recorded_at=now). Return the new row id.
       - `def get_metric(conn, name: str) -> MetricDefinition | None`: SELECT from `current_derived_metrics` WHERE metric_name=name. Return MetricDefinition or None.
       - `def get_all_metrics(conn) -> list[MetricDefinition]`: SELECT all from `current_derived_metrics`.
       - `def get_schema_version(conn) -> int`: Return MAX(recorded_at_ts) across db_tables, db_columns, db_relationships tables. Return 0 if all empty.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> define_metric | User-supplied SQL fragments could contain malicious SQL |
| NL question -> SQL generation | Natural language input influences generated SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Injection | resolver.py:define_metric | mitigate | Validate SQL fragment with sqlglot.parse_one(); reject fragments containing ;, DROP, INSERT, DELETE, EXEC, UPDATE keywords. Only accept expressions, not full statements. |
| T-04-02 | Tampering | wiki.py:wiki_pages cache | accept | Cache is local SQLite, same trust boundary as knowledge store. Stale cache returns outdated info but not harmful — invalidation via schema_version. |
</threat_model>
